# OFD Reader 轻阅读 - PRD 修改计划

## 1. 需求分析

### 1.1 原始需求
- Web端OFD文档阅读器，提供完整的阅读功能
- 支持用户注册登录、阅读历史、个性化设置等
- 面向普通用户、政府、企业等多种用户群体

### 1.2 新需求变更
- **使用场景**：保险公司发送OFD电子保单给用户时，附带网页链接直接阅读
- **目标客户**：保险公司（B端），而非终端用户（C端）
- **功能范围**：移除首页、个人中心、用户管理等功能，仅保留核心阅读功能
- **合规要求**：满足OFD电子保单行业规范的阅读要求

## 2. 修改内容

### 2.1 文档结构调整
- 更新版本记录，增加V1.1版本
- 修改需求背景与产品目标，聚焦保险公司使用场景
- 调整目标用户，明确为保险公司
- 简化产品范围，移除非核心功能
- 更新业务流程图，反映新的使用流程
- 调整页面设计，移除不需要的页面

### 2.2 具体修改点

#### 2.2.1 需求背景与产品目标
- 修改需求背景，强调保险公司发送电子保单的场景
- 更新产品目标，聚焦于为保险公司提供轻量级阅读工具
- 调整量化价值，从保险公司角度出发

#### 2.2.2 产品范围
- 核心功能：仅保留OFD文档阅读、页面导航、缩放、搜索等基本阅读功能
- 移除扩展功能：用户注册登录、阅读历史记录、个性化设置、文档云存储
- 技术范围：保持Web端应用，支持主流浏览器，响应式设计

#### 2.2.3 业务流程
- 简化业务流程，移除登录、注册、个人中心等环节
- 新流程：保险公司生成带阅读器链接的保单 → 用户点击链接 → 直接进入阅读器页面 → 阅读保单 → 完成

#### 2.2.4 功能描述
- 移除文档上传功能（由保险公司直接提供文档）
- 移除用户管理相关功能
- 保留并强化阅读器核心功能
- 增加保单阅读相关的特定功能（如关键字高亮、重要条款标记等）

#### 2.2.5 界面设计
- 移除首页、个人中心等页面
- 优化阅读器页面，专注于阅读体验
- 调整设计风格，符合保险公司品牌形象

#### 2.2.6 技术要求
- 保持前端技术栈不变
- 移除后端相关技术要求（无需用户管理和云存储）
- 强调性能和安全性，确保保单信息安全

## 3. 实施步骤

1. **更新版本记录**：添加V1.1版本，记录变更内容
2. **修改需求背景与产品目标**：调整为保险公司使用场景
3. **更新产品范围**：移除不需要的功能，聚焦核心阅读功能
4. **重绘业务流程图**：反映新的使用流程
5. **简化功能描述**：移除用户管理等相关功能
6. **调整界面设计**：移除不需要的页面，优化阅读器界面
7. **更新技术要求**：移除后端相关要求，强调前端性能和安全
8. **添加行业规范参考**：包含OFD电子保单行业规范的阅读要求

## 4. 风险评估

### 4.1 潜在风险
- **功能简化风险**：移除部分功能可能影响用户体验
- **行业规范合规风险**：需要确保满足OFD电子保单的行业规范
- **技术实现风险**：需要确保OFD解析的准确性和性能

### 4.2 缓解措施
- **功能优先级**：确保核心阅读功能完善，满足基本阅读需求
- **合规性检查**：参考OFD电子保单行业规范，确保阅读功能符合要求
- **技术选型**：选择成熟的OFD解析库，确保兼容性和性能

- 一份针对保险公司使用场景的PRD文档
- 聚焦于核心阅读功能的产品设计
- 符合OFD电子保单行业规范的阅读体验
- 轻量级、易于集成的Web端阅读器解决方案
